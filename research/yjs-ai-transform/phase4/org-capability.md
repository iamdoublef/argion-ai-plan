# 组织能力建设方案

> Phase 4 产出 | 项目：亚俊氏AI转型研究
> 日期：2026-02-14 | 更新：2026-02-14（v2.0 强化组织变革管理，引入BCG 10-20-70法则）

## 方法论基础

本方案参考以下方法论：
- **BCG "10-20-70"法则**：AI转型成功的关键投入分配为10%技术、20%数据、70%组织变革（来源：BCG AI@Scale研究）
- **McKinsey "Rewired"方法论**：AI转型不是技术项目，而是企业"重新布线"，需要系统化的变革管理（来源：McKinsey Rewired）
- **PwC Strategy& 九步框架**：第7步"运营模式转型"强调组织和流程调整（来源：PwC Strategy&）

> **核心洞察**：BCG研究表明，AI转型失败的首要原因不是技术，而是组织变革不到位。70%的精力应放在人员、流程、文化的变革上。这意味着我们的组织能力建设不是"配套工作"，而是AI转型的核心工作。

## 一、变革管理框架（新增）

### 1.1 变革管理三层模型

```
┌─────────────────────────────────────────────┐
│           第三层：文化变革（持续）              │
│  数据驱动决策文化 → AI优先思维 → 持续学习文化   │
├─────────────────────────────────────────────┤
│           第二层：流程变革（6-18月）            │
│  业务流程AI化 → 决策流程数据化 → 协作模式转变   │
├─────────────────────────────────────────────┤
│           第一层：认知变革（0-6月）             │
│  管理层AI认知 → 全员AI意识 → 消除恐惧和抵触    │
└─────────────────────────────────────────────┘
```

### 1.2 变革管理关键举措

| 阶段 | 举措 | 负责人 | 预期效果 |
|------|------|--------|---------|
| 0-3月 | CEO发布AI转型愿景信，明确"AI赋能不裁员"承诺 | CEO | 消除员工恐惧 |
| 0-3月 | 管理层AI认知培训（含制造业AI案例研讨） | 外部顾问 | 管理层理解AI价值 |
| 3-6月 | 设立"AI创新奖"，鼓励员工提出AI应用建议 | AI项目经理 | 激发全员参与 |
| 3-6月 | 每月AI进展通报（全员可见） | AI项目经理 | 透明沟通，建立信任 |
| 6-12月 | 速赢项目成果展示会（让全员看到AI的实际效果） | 各部门AI联络人 | 用事实说话 |
| 6-18月 | 业务流程AI化改造（从"人做"到"人+AI做"） | 各部门负责人 | 流程变革落地 |
| 12-24月 | 建立数据驱动决策机制（关键决策必须有数据支撑） | CEO | 文化变革 |
| 18-36月 | AI能力纳入绩效考核和晋升标准 | HR | 制度化保障 |

### 1.3 投入分配对标BCG 10-20-70

| 类别 | BCG建议占比 | 我们的预算分配 | 调整建议 |
|------|:---:|:---:|---------|
| 技术（AI工具、硬件、平台） | 10% | ~49%（硬件+软件+集成） | 技术投入占比偏高，但制造业AI质检等硬件投入不可避免 |
| 数据（数据治理、数据平台） | 20% | ~13%（数据治理+平台） | 建议适当增加数据投入 |
| 组织变革（人才、培训、变革管理） | 70% | ~38%（人力+培训+顾问） | 需要增加变革管理专项投入 |

> **说明**：制造业AI转型因涉及硬件（工业相机、IoT传感器等），技术投入占比天然高于纯软件行业。但组织变革投入仍需加强，建议在预备金中划拨30-50万专项用于变革管理活动。

## 二、AI团队架构

### 总体策略：混合模式（自建核心 + 外部协作）

考虑亚俊氏的规模（350人、2.92亿营收）和薪酬水平（4.5-15K），不适合全自建AI团队。建议采用"小而精的核心团队 + 外部生态"模式。

### 团队演进路径

```
阶段一（0-6月）         阶段二（6-18月）        阶段三（18-36月）
1人核心 + 外部顾问      3人核心 + 外部实施      5人核心 + 外部合作
                        + 内部转岗培养          + 高校联培

┌─────────────┐    ┌──────────────────┐    ┌────────────────────────┐
│ AI项目经理   │    │ AI项目经理        │    │ AI负责人（升级）        │
│ (1人)       │    │ 数据工程师(1人)   │    │ AI产品经理(1人)        │
│             │    │ AI应用工程师(1人) │    │ 数据工程师(1-2人)      │
│ + 外部顾问   │    │                  │    │ AI应用工程师(1-2人)    │
│   (按需)    │    │ + 外部实施团队     │    │                        │
└─────────────┘    └──────────────────┘    │ + 外部合作伙伴          │
                                           │ + 高校联培学生           │
                                           └────────────────────────┘
```

### 组织架构建议

```
CEO 梁国强（AI转型总负责人）
│
├── AI转型办公室（虚拟组织，阶段一二）→ 数字化与AI部（实体部门，阶段三）
│   ├── AI负责人/项目经理（汇报给CEO）
│   ├── 数据工程师
│   └── AI应用工程师
│
├── 研发中心（AI设计工具使用者）
├── 生产运营（AI质检、排产使用者）
├── 销售市场（AI内容、电商工具使用者）
└── 供应链（AI预测使用者）
```

**关键设计原则：**
- AI团队直接向CEO汇报，避免被某个业务部门"吞没"
- 阶段一二为虚拟组织（跨部门协调），阶段三升级为实体部门
- 每个业务部门设1名"AI联络人"，负责本部门AI需求对接

## 二、关键岗位定义

### 岗位 1：AI项目经理 / AI负责人

| 项目 | 内容 |
|------|------|
| 到岗时间 | 第1-2个月（最优先） |
| 职级 | 中高层（向CEO直接汇报） |
| 薪酬范围 | 25-40万/年（需突破现有薪酬体系） |
| 核心职责 | 统筹AI转型项目规划与执行；供应商选型与管理；内部AI能力建设；向管理层汇报进展 |
| 能力要求 | 3年以上AI项目管理经验；有制造业AI落地经验优先；懂技术但更懂业务；沟通协调能力强 |
| 招聘渠道 | 猎头、LinkedIn、AI行业社群 |
| 可接受远程 | 是（扩大招聘范围） |

### 岗位 2：数据工程师

| 项目 | 内容 |
|------|------|
| 到岗时间 | 第3-6个月 |
| 职级 | 中级 |
| 薪酬范围 | 15-25万/年 |
| 核心职责 | 数据治理与数据平台建设；ETL流程开发；数据质量监控；支撑AI模型的数据准备 |
| 能力要求 | 熟悉SQL、Python、ETL工具；有ERP数据对接经验优先；了解数据仓库架构 |
| 招聘渠道 | 招聘网站、校招（华南理工等） |
| 替代方案 | 从现有IT人员中培养（如有） |

### 岗位 3：AI应用工程师

| 项目 | 内容 |
|------|------|
| 到岗时间 | 第6-9个月 |
| 职级 | 中级 |
| 薪酬范围 | 18-30万/年 |
| 核心职责 | AI模型调优与部署；AI SaaS工具集成；内部AI工具开发；AI系统运维 |
| 能力要求 | 熟悉Python、ML框架；有计算机视觉或NLP经验优先；能独立完成模型训练和部署 |
| 招聘渠道 | 招聘网站、AI社群、高校联培 |
| 替代方案 | 从研发工程师中选拔培养 |

### 岗位 4：AI产品经理（阶段三）

| 项目 | 内容 |
|------|------|
| 到岗时间 | 第18-24个月 |
| 职级 | 中级 |
| 薪酬范围 | 20-35万/年 |
| 核心职责 | 智能产品（IoT+AI）的产品规划；App和云端服务设计；用户体验设计 |
| 能力要求 | 有IoT或智能硬件产品经验；懂AI能力边界；用户思维强 |
| 招聘渠道 | 猎头、智能硬件行业 |

### 岗位 5：各部门AI联络人（兼职）

| 项目 | 内容 |
|------|------|
| 到岗时间 | 第2-3个月（内部选拔） |
| 人数 | 4-5人（研发、生产、销售、供应链、客服各1人） |
| 薪酬 | 现有薪酬 + AI项目津贴（500-1000元/月） |
| 核心职责 | 收集本部门AI需求；协调本部门配合AI项目；推动AI工具在本部门的使用 |
| 选拔标准 | 对新技术有兴趣；在部门内有影响力；沟通能力好 |

## 三、培训计划

### 3.1 管理层AI认知培训

| 项目 | 内容 |
|------|------|
| 对象 | CEO + 核心管理层（5-8人） |
| 时间 | 第1-2个月，共2天 |
| 形式 | 外部AI顾问授课 + 制造业AI案例研讨 |
| 费用 | 3-5万 |

**课程内容：**

| 模块 | 时长 | 内容 |
|------|------|------|
| AI基础认知 | 半天 | AI/ML/LLM基本概念；AI能做什么、不能做什么；制造业AI现状 |
| 制造业AI案例 | 半天 | 同行业AI转型案例（美的、海尔等大企业简化版）；中小企业AI落地案例 |
| 亚俊氏AI机会 | 半天 | 基于Phase 1-3研究成果的定制化分析；各业务环节AI场景讨论 |
| 战略研讨 | 半天 | AI转型路线图讨论；资源分配决策；Go/No-Go机制确认 |

### 3.2 业务人员AI工具培训

| 项目 | 内容 |
|------|------|
| 对象 | 市场/销售团队、研发设计师、客服人员（共30-50人） |
| 时间 | 第2-4个月，分批进行 |
| 形式 | 内部培训 + 实操练习 |
| 费用 | 2-3万 |

**课程内容（按角色定制）：**

| 角色 | 培训内容 | 时长 |
|------|---------|------|
| 市场/电商团队 | ChatGPT/Claude使用；Prompt工程基础；AI图片生成工具；电商AI工具 | 2天 |
| 研发设计师 | AI图像生成（Midjourney等）；AI辅助设计流程；Prompt工程进阶 | 2天 |
| 客服人员 | AI客服系统使用；知识库维护；人机协作流程 | 1天 |

### 3.3 技术人员AI开发培训

| 项目 | 内容 |
|------|------|
| 对象 | 内部转岗候选人（3-5人，从研发工程师中选拔） |
| 时间 | 第3-9个月，持续学习 |
| 形式 | 在线课程 + 项目实战 + 外部导师指导 |
| 费用 | 3-5万 |

**培训路径：**

```
第3-4月：基础阶段
├── Python编程强化（如已有基础则跳过）
├── 数据分析基础（Pandas、SQL）
└── ML基础概念

第5-6月：进阶阶段
├── 机器学习实战（scikit-learn）
├── 深度学习基础（PyTorch）
└── 计算机视觉或NLP（根据方向选择）

第7-9月：实战阶段
├── 参与AI质检项目（跟随供应商学习）
├── 参与需求预测模型开发
└── 独立完成小型AI应用
```

**推荐学习资源：**
- 吴恩达 Machine Learning 课程（Coursera）
- fast.ai 实战课程
- 百度飞桨/华为昇思 制造业AI课程（中文）

### 3.4 全员AI意识普及

| 项目 | 内容 |
|------|------|
| 对象 | 全体员工（350人） |
| 时间 | 第4-6个月 |
| 形式 | 线上视频课程 + 线下宣讲会 |
| 费用 | 1-2万 |

**内容：**
- AI是什么（30分钟视频）
- AI如何改变我们的工作（案例分享）
- 亚俊氏AI转型计划简介
- 常见问题解答（AI会不会替代我的工作？）

## 四、外部合作伙伴

### 4.1 合作伙伴类型

| 类型 | 角色 | 合作模式 | 预算 |
|------|------|---------|------|
| AI战略顾问 | 转型规划、项目监理 | 按项目/按天计费 | 10-20万/年 |
| AI视觉质检供应商 | 硬件+软件+部署 | 项目制 | 见预算估算 |
| AI SaaS工具商 | 提供现成AI工具 | 订阅制 | 15-30万/年 |
| 系统集成商 | ERP/MES集成、数据平台 | 项目制 | 40-80万 |
| 高校/研究机构 | 人才培养、技术合作 | 联合项目/实习 | 5-10万/年 |

### 4.2 推荐合作伙伴方向

**AI战略顾问（选1家）：**
- 德勤/埃森哲的中小企业AI咨询团队
- 国内AI咨询公司（如亿欧智库、36氪研究院等）
- 独立AI顾问（有制造业经验的）

**AI视觉质检供应商（评估3家，选1家）：**
- 百度智能云 — 工业视觉质检平台（生态完善）
- 海康威视 — 机器视觉+AI检测（硬件优势）
- 创新奇智 — 制造业AI质检（专注制造业）

**高校合作（选1-2家）：**
- 华南理工大学 — 计算机/自动化学院（地理优势，广州）
- 中山大学 — 智能工程学院
- 广东工业大学 — 自动化学院（实用型人才）

### 4.3 合作管理原则

1. 核心知识产权归亚俊氏所有（合同明确约定）
2. 数据不出厂（敏感数据在本地处理）
3. 逐步建立内部能力，减少对外部的依赖
4. 每个外部合作项目必须有内部对接人跟进学习
5. 供应商评估每年一次，保持竞争性
