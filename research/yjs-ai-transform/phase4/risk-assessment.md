# 风险评估

> Phase 4 产出 | 项目：亚俊氏AI转型研究
> 日期：2026-02-14

## 风险评估框架

**概率等级：** 高（>60%）、中（30-60%）、低（<30%）
**影响等级：** 高（可能导致项目失败或重大延期）、中（导致部分目标未达成）、低（可控偏差）
**风险等级 = 概率 × 影响**

---

## 风险矩阵总览

```
         │  低影响    │  中影响    │  高影响
─────────┼───────────┼───────────┼──────────
高概率   │           │  R3 人才   │  R2 数据
         │           │  R4 组织   │
─────────┼───────────┼───────────┼──────────
中概率   │  R7 供应商 │  R5 资金   │  R1 技术
         │           │  R8 合规   │
─────────┼───────────┼───────────┼──────────
低概率   │           │  R6 市场   │
         │           │           │
```

## 风险详细评估

### R1：技术风险 — AI方案效果不达预期

| 项目 | 内容 |
|------|------|
| 概率 | 中（40%） |
| 影响 | 高 |
| 风险等级 | 高 |
| 描述 | AI视觉质检准确率不达标；需求预测模型效果不佳；智能产品技术方案不成熟 |
| 受影响项目 | 2.1 AI质检、2.2 需求预测、3.3 智能产品 |
| 最坏情况 | 核心AI项目无法落地，投入打水漂，动摇管理层信心 |

**应对措施：**

| 措施 | 具体行动 | 负责人 | 时间 |
|------|---------|--------|------|
| 预防 | 先做小规模PoC（概念验证），验证后再全面推广 | AI项目经理 | 每个项目启动前 |
| 预防 | 选择成熟度高的技术方案，优先使用已有成功案例的供应商 | AI项目经理 | 供应商选型阶段 |
| 缓解 | AI质检先试点2-3条线，设置3个月验证期 | 生产运营 | 第7-9月 |
| 缓解 | 需求预测先用简单模型（如Prophet），逐步迭代 | 数据工程师 | 第7-9月 |
| 应急 | 准备备选供应商名单，技术方案不行时快速切换 | AI项目经理 | 持续 |
| 应急 | 设置止损线：单项目PoC阶段投入不超过总预算20% | CEO | 每个Go/No-Go点 |

### R2：数据风险 — 数据质量不足、数据孤岛难打通

| 项目 | 内容 |
|------|------|
| 概率 | 高（70%） |
| 影响 | 高 |
| 风险等级 | 极高 |
| 描述 | 现有系统数据质量差（缺失、不一致、格式混乱）；ERP等系统数据导出困难；各系统间数据无法打通 |
| 受影响项目 | 1.4 数据治理、2.2 需求预测、2.5 数据平台、3.1 智能排产 |
| 最坏情况 | 数据基础不具备，依赖数据的AI项目全部延期或失败 |

**应对措施：**

| 措施 | 具体行动 | 负责人 | 时间 |
|------|---------|--------|------|
| 预防 | 阶段一优先启动数据治理，不等AI项目 | IT负责人 | 第1月起 |
| 预防 | 聘请有制造业经验的数据顾问，避免走弯路 | AI项目经理 | 第1月 |
| 缓解 | 分模块推进：先打通订单-库存数据，再扩展到生产数据 | 数据工程师 | 第1-12月 |
| 缓解 | 对于数据质量差的场景，先用规则引擎+简单统计，再逐步引入ML | 数据工程师 | 持续 |
| 应急 | 如果ERP数据导出困难，考虑升级ERP或引入中间件 | IT负责人 | 第6月评估 |
| 应急 | 速赢项目（内容生成、辅助设计）不依赖内部数据，确保这些项目不受影响 | AI项目经理 | 持续 |

### R3：人才风险 — 招不到/留不住AI人才

| 项目 | 内容 |
|------|------|
| 概率 | 高（65%） |
| 影响 | 中 |
| 风险等级 | 高 |
| 描述 | 亚俊氏薪酬水平（4.5-15K）远低于AI人才市场价（20-50K）；广州/佛山非AI人才聚集地；制造业对AI人才吸引力有限 |
| 受影响项目 | 所有需要内部AI能力的项目 |
| 最坏情况 | 无法组建内部AI团队，完全依赖外部，成本高且知识无法沉淀 |

**应对措施：**

| 措施 | 具体行动 | 负责人 | 时间 |
|------|---------|--------|------|
| 预防 | 采用"1+N"模式：1名核心AI负责人（高薪）+ N名培养型人才（内部转岗） | HR + CEO | 第1-3月 |
| 预防 | AI项目经理可接受远程办公，扩大招聘范围 | HR | 第1月起 |
| 缓解 | 从现有100名研发工程师中选拔3-5人转型为AI应用工程师 | 研发中心 | 第2-6月 |
| 缓解 | 与高校合作（华南理工、中山大学等），建立实习/联培机制 | HR | 第3月起 |
| 应急 | 阶段一二以外包为主，内部团队逐步接管 | AI项目经理 | 持续 |
| 应急 | 考虑股权激励或项目奖金吸引核心AI人才 | CEO + 财务 | 第3月评估 |

### R4：组织风险 — 员工抵触、部门协作困难

| 项目 | 内容 |
|------|------|
| 概率 | 高（60%） |
| 影响 | 中 |
| 风险等级 | 高 |
| 描述 | 一线员工担心AI替代岗位；部门间数据共享意愿低；中层管理者对AI项目配合度不足 |
| 受影响项目 | 2.1 AI质检（质检员抵触）、2.5 数据平台（部门不配合）、3.1 智能排产（生产主管抵触） |
| 最坏情况 | AI项目推进受阻，数据获取困难，系统上线后无人使用 |

**应对措施：**

| 措施 | 具体行动 | 负责人 | 时间 |
|------|---------|--------|------|
| 预防 | CEO亲自担任AI转型总负责人，发出明确信号 | CEO | 第1月 |
| 预防 | 明确承诺：AI替代的岗位优先转岗而非裁员 | CEO + HR | 第1月 |
| 缓解 | 将AI项目KPI纳入部门考核，与奖金挂钩 | HR | 第3月起 |
| 缓解 | 每月举办AI成果分享会，展示实际效益 | AI项目经理 | 第3月起 |
| 缓解 | 选择配合度高的部门/产线先试点，树立标杆 | AI项目经理 | 每个项目启动时 |
| 应急 | 对于强烈抵触的部门，暂缓推进，先做其他部门 | CEO | 持续评估 |

### R5：资金风险 — 投入超预期、ROI不达标

| 项目 | 内容 |
|------|------|
| 概率 | 中（45%） |
| 影响 | 中 |
| 风险等级 | 中 |
| 描述 | AI项目实施成本超出预算；收益兑现慢于预期；经济环境变化导致资金紧张 |
| 受影响项目 | 所有项目，尤其是阶段二三的大型项目 |
| 最坏情况 | 资金链紧张，被迫中止部分AI项目 |

**应对措施：**

| 措施 | 具体行动 | 负责人 | 时间 |
|------|---------|--------|------|
| 预防 | 每个阶段预留15%预备金 | 财务 | 预算编制时 |
| 预防 | 阶段一优先选择低投入高回报的速赢项目 | AI项目经理 | 第1月 |
| 缓解 | 按季度审查预算执行情况，及时调整 | 财务 + AI项目经理 | 每季度 |
| 缓解 | 大型项目（AI质检、智能产品）分期投入，每期验收后再追加 | AI项目经理 | 持续 |
| 应急 | 如资金紧张，优先保障已见效项目的运营，暂缓新项目 | CEO | Go/No-Go点 |
| 应急 | 探索政府AI补贴和专项资金（广东省智能制造补贴等） | 财务 | 第1月起 |

### R6：市场风险 — 行业变化导致方向调整

| 项目 | 内容 |
|------|------|
| 概率 | 低（20%） |
| 影响 | 中 |
| 风险等级 | 低-中 |
| 描述 | 真空厨房设备市场萎缩；主要ODM客户流失；贸易政策变化影响出口 |
| 受影响项目 | 3.3 智能产品（市场需求变化）、整体转型节奏 |
| 最坏情况 | 市场环境恶化，AI投入的优先级被迫让位于生存问题 |

**应对措施：**

| 措施 | 具体行动 | 负责人 | 时间 |
|------|---------|--------|------|
| 预防 | 项目2.4市场趋势监控提供早期预警 | 市场部 | 第9月起 |
| 缓解 | AI能力建设（数据平台、AI团队）具有通用性，即使方向调整也不浪费 | AI项目经理 | 持续 |
| 缓解 | 保持路线图灵活性，每个Go/No-Go点可调整方向 | CEO | 每6个月 |
| 应急 | 如市场剧变，收缩到已验证的高ROI项目，暂停探索性项目 | CEO | 触发时 |

### R7：供应商风险 — AI供应商选择不当或服务不达标

| 项目 | 内容 |
|------|------|
| 概率 | 中（35%） |
| 影响 | 低-中 |
| 风险等级 | 中 |
| 描述 | AI供应商技术能力不足；供应商倒闭或停止服务；被供应商锁定 |
| 受影响项目 | 2.1 AI质检（硬件+软件供应商）、SaaS工具类项目 |
| 最坏情况 | 供应商无法交付，项目延期；更换供应商导致重复投入 |

**应对措施：**

| 措施 | 具体行动 | 负责人 | 时间 |
|------|---------|--------|------|
| 预防 | 每个项目至少评估3家供应商，选择有制造业案例的 | AI项目经理 | 供应商选型阶段 |
| 预防 | 合同中约定SLA、数据归属权、退出条款 | 法务 + AI项目经理 | 签约前 |
| 缓解 | 避免深度绑定单一供应商，核心数据和模型保留在自有环境 | IT | 持续 |
| 应急 | 维护备选供应商名单，确保可在3个月内切换 | AI项目经理 | 持续 |

### R8：合规与安全风险 — 数据安全和知识产权问题

| 项目 | 内容 |
|------|------|
| 概率 | 中（30%） |
| 影响 | 中 |
| 风险等级 | 中 |
| 描述 | AI工具使用中泄露客户机密（ODM设计图纸）；AI生成内容的知识产权归属不清；数据跨境传输合规问题（新加坡基地） |
| 受影响项目 | 1.1 内容生成、1.3 辅助设计、2.3 AI客服 |
| 最坏情况 | 客户机密泄露导致ODM客户流失；知识产权纠纷 |

**应对措施：**

| 措施 | 具体行动 | 负责人 | 时间 |
|------|---------|--------|------|
| 预防 | 制定AI使用安全规范：禁止上传客户机密到公共AI服务 | IT + 法务 | 第1月 |
| 预防 | 敏感数据使用私有化部署的AI模型 | IT | 第3月起 |
| 缓解 | AI生成的设计图仅作参考，最终设计由人工完成并确认版权 | 研发中心 | 持续 |
| 缓解 | 与法务确认AI生成内容的知识产权归属 | 法务 | 第2月 |
| 应急 | 发现数据泄露立即停用相关AI工具，启动应急响应 | IT | 触发时 |

## 风险监控机制

### 定期评审

| 频率 | 内容 | 参与人 |
|------|------|--------|
| 每月 | 项目进度和风险状态更新 | AI项目经理 + 项目组 |
| 每季度 | 风险矩阵重新评估，更新应对措施 | CEO + 核心管理层 |
| Go/No-Go点 | 全面风险评估，决定是否进入下一阶段 | CEO + 全体管理层 |

### 风险预警指标

| 预警指标 | 黄灯（关注） | 红灯（行动） |
|----------|-------------|-------------|
| 项目进度 | 延期 > 2周 | 延期 > 1个月 |
| 预算执行 | 超支 > 10% | 超支 > 20% |
| AI模型效果 | 低于目标10% | 低于目标20% |
| 人才流失 | 核心成员提出离职 | 核心成员已离职 |
| 数据质量 | 数据完整率 < 80% | 数据完整率 < 60% |
