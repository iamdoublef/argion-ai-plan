# AI转型路线图（36个月）

> Phase 4 产出 | 项目：亚俊氏AI转型研究
> 日期：2026-02-14 | 更新：2026-02-14（v2.0 对齐CMMM成熟度目标+AI治理+变革管理）

## 方法论基础

本路线图参考以下方法论组合（详见 `audit/methodology-review.md`）：
- **CMMM（GB/T 39116-2020）**：成熟度评估与阶段目标设定
- **PwC Strategy& 九步框架**：制造业AI转型路径（就绪度评估→PoC→试点→规模化）
- **Microsoft 5阶段AI成熟度**：阶段目标对标
- **BCG "10-20-70"法则**：投入分配指导（10%技术、20%数据、70%组织变革）

## 总体规划

```
阶段一（0-6月）        阶段二（6-18月）          阶段三（18-36月）
基础建设 + 速赢        核心场景深化              全面智能化
CMMM L2→L2.5          CMMM L2.5→L3             CMMM L3→L3.5-L4
─────────────────────────────────────────────────────────────
数据治理启动 ──────→  数据平台建成 ──────────→  数据驱动决策体系
AI SaaS工具上线 ──→  生产AI落地 ────────────→  供应链端到端智能
AI意识培训 ────────→  AI团队扩充 ────────────→  产品智能化
认知变革 ─────────→  流程变革 ──────────────→  文化变革+AI治理
AI就绪度基线评估 ──→  参与CMMM官方评估 ────→  AI驱动新业务
```

### 成熟度阶段目标

| 时间节点 | CMMM目标 | Microsoft AI目标 | 关键标志 |
|----------|---------|-----------------|---------|
| 第6月末 | L2.5 | Foundational→Approaching | 数据资产盘点完成；3个AI SaaS工具在用 |
| 第18月末 | L3（集成级） | Approaching→Aspirational | 核心系统数据打通；AI质检产线覆盖；BI看板上线 |
| 第36月末 | L3.5-L4（优化级） | Aspirational→Mature | 数据驱动决策；多AI应用协同；智能产品上市 |

| 指标 | Go条件 | No-Go处置 |
|------|--------|-----------|
| AI质检效果 | 试点线准确率 > 99%，已开始推广 | 更换供应商或调整技术方案 |
| 需求预测 | 预测准确率提升 > 15% | 评估数据质量，考虑增加数据源 |
| 数据平台 | 至少3个核心系统数据打通 | 加大系统集成投入 |
| CMMM评估 | 获得L2+认证，L3差距明确 | 聚焦L3差距项补强 |
| 累计ROI | 阶段一+二项目累计ROI > 150% | 收缩阶段三范围，聚焦已验证场景 |
| AI团队 | 核心AI团队 ≥ 3人 | 加大招聘力度或增加外包比例 |
| 变革管理 | 业务流程AI化改造启动，员工接受度良好 | 加强变革管理和沟通 |

**AI治理框架要点**（参考Deloitte可信AI框架）：

| 维度 | 亚俊氏适用要求 |
|------|---------------|
| 数据安全 | 客户数据、产品数据不外泄；敏感数据本地化处理 |
| 透明可解释 | AI质检判定结果可追溯；AI预测依据可查看 |
| 公平性 | AI招聘/绩效工具（如有）需确保无偏见 |
| 知识产权 | AI生成内容的版权归属明确；AI辅助设计的专利归属明确 |
| 合规性 | AI产品符合出口目的地法规（EU AI Act等） |
