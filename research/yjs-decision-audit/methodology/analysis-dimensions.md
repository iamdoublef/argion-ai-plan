## 维度A：错误识别与分类

### A1. 错误分类（5类）
| 代码 | 类型 | 定义 | 典型场景 |
|------|------|------|---------|
| ERR-OMI | 遗漏错误 | 该考虑的选项/因素未考虑 | 库存决策未考虑"销毁" |
| ERR-COM | 执行错误 | 选择了错误的方案或执行偏差 | 说明书用公制而非英制 |
| ERR-REC | 重复错误 | 同一人同类错误再次发生 | 欧洲市场重复库存错误 |
| ERR-PRC | 流程错误 | 决策/审核流程本身有缺陷 | 多层审核均未发现问题 |
| ERR-DLY | 延迟错误 | 决策时机过晚导致损失扩大 | 未及时换人/调整策略 |

### A2. 错误分级（3级 + 重复加权）
| 等级 | 判定标准 | RPN参考 |
|------|---------|---------|
| 严重（Critical） | 经济损失>5万 或 品牌/法规风险 | RPN≥200 |
| 次要（Major） | 可衡量的效率/体验损失 | RPN 80-199 |
| 一般（Minor） | 轻微影响，可快速修复 | RPN<80 |

重复加权规则：
- 6个月内重复 → 等级+1
- 12个月内第3次 → 直接标记严重

### A3. 错误指纹（用于重复识别）
```
ErrorFingerprint {
  责任人ID
  错误类型（5类之一）
  业务领域（库存/产品/市场/供应链/文档/人事）
  根因分类（认知偏差类型）
  发生日期
}
```
匹配规则：责任人 + 错误类型 + 业务领域 三维匹配 → 标记重复

## 维度B：责任追溯与分配

### B1. 三层责任模型（Swiss Cheese）
```
L3 决策层（副总）     ──── 管理监督责任
L2 管理层（部门经理） ──── 审核把关责任
L1 执行层（工程师）   ──── 直接执行责任
```

### B2. 责任权重（动态调整）

| 场景 | L1执行 | L2管理 | L3决策 |
|------|--------|--------|--------|
| 首次错误 | 40% | 35% | 25% |
| 同一人第2次 | 30% | 35% | 35% |
| 同一人第3次+ | 20% | 30% | 50% |
| 副总本人决策错误 | — | — | 100% |

### B3. 业务场景覆盖
| 场景 | L1 | L2 | L3 |
|------|----|----|-----|
| 产品说明书 | 工程师（编写） | 经理（审核） | 副总（分管） |
| 库存决策 | 运营专员（执行） | 运营经理（方案） | 副总（决策） |
| 市场策略 | 市场专员（执行） | 市场经理（策划） | 副总（审批） |
| 供应链 | 采购员（执行） | 采购经理（选型） | 副总（审批） |
| 人事决策 | HR（执行） | 部门经理（提议） | 副总（决定） |

## 维度C：考核联动与激励

### C1. 各层级KPI指标

**执行层**
| 指标 | 权重 | 数据来源 |
|------|------|---------|
| 个人错误分值 | 50% | Σ(错误等级×权重) |
| 重复错误率 | 30% | 重复数/总数 |
| 改进率 | 20% | 环比变化 |

**管理层**
| 指标 | 权重 | 数据来源 |
|------|------|---------|
| 团队错误总分 | 30% | 团队Σ(错误分值) |
| 审核拦截率 | 30% | 拦截数/总错误数 |
| 下属重复率 | 20% | 下属重复错误占比 |
| 团队改进趋势 | 20% | 季度环比 |

**决策层**
| 指标 | 权重 | 数据来源 |
|------|------|---------|
| 重大错误数 | 25% | 严重级错误次数 |
| 个人决策失误 | 25% | 本人决策错误 |
| 管理有效性 | 25% | 下属整体趋势 |
| 人员管理 | 25% | 是否及时处理问题 |

### C2. 薪酬联动公式

```
年终奖 = 基准奖金 × 绩效系数

绩效系数 = 1.0 - Σ(错误扣分)

扣分标准：
  严重错误：-0.15/次
  次要错误：-0.04/次
  一般错误：-0.01/次
  重复加倍：扣分×2

绩效系数范围：[0.3, 1.2]
  ≥1.0 → 有资格加薪
  0.8-0.99 → 薪酬不变
  0.6-0.79 → 警告
  <0.6 → 降薪/调岗评估
```

## 基础设施需求

### 数据采集层
| 数据源 | 采集方式 | 频率 |
|--------|---------|------|
| 文档错误 | AI自动扫描（说明书、报告等） | 实时/批量 |
| 决策记录 | 会议纪要录入/语音转文字 | 每次会议 |
| 错误报告 | 人工录入+AI辅助分类 | 发现即录 |
| 执行偏差 | 与ERP/OA系统对接 | 定期同步 |

### AI引擎能力
| 能力 | 用途 |
|------|------|
| 文档核查 | 自动扫描说明书等文档的错漏 |
| 错误分类 | 基于描述自动归类错误类型 |
| 重复识别 | 错误指纹匹配，识别重复模式 |
| 趋势分析 | 个人/团队错误趋势可视化 |
| 报告生成 | 自动生成年度考核报告 |

## 实施路径建议

```
Phase 1（1-2月）：制度建设
  → 定义错误分类标准和分级标准
  → 制定考核制度，经职工代表大会通过
  → 员工签署知情同意书

Phase 2（2-3月）：MVP系统
  → 错误记录表单（可以是简单的在线表格）
  → 手动录入+基础统计
  → 月度报告

Phase 3（3-6月）：AI增强
  → 文档自动核查（说明书扫描）
  → 错误指纹匹配（重复识别）
  → 自动生成考核报告

Phase 4（6-12月）：深度集成
  → 与OA/HR系统对接
  → 决策会议记录系统
  → 执行偏差追踪
```
