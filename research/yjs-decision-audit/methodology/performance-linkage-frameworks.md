## 1. 主流绩效考核体系中纳入"决策质量"

### 1.1 KPI（关键绩效指标）
- 传统KPI侧重结果指标（营收、利润、交付率）
- 纳入决策质量的方式：增加"过程质量KPI"
  - 决策错误次数（按严重度加权）
  - 重复错误率（同类错误再发次数/总决策数）
  - 审核拦截率（经理/副总层面拦截的错误占比）
- ⚠️ 亚俊氏适用性：KPI简单直接，适合制造业文化，管理者容易理解

### 1.2 BSC（平衡计分卡）
- 四个维度：财务、客户、内部流程、学习与成长
- 决策质量可嵌入"内部流程"维度：
  - 指标：决策错误率、重复错误率、决策审计通过率
- 也可嵌入"学习与成长"维度：
  - 指标：错误改进率（同类错误是否减少）、决策能力提升评估
- ⚠️ 亚俊氏适用性：BSC更全面但实施复杂，建议先用KPI，成熟后升级到BSC

### 1.3 OKR（目标与关键结果）
- 不适合直接用于惩罚性考核（OKR设计理念是激励而非问责）
- 但可用于设定改进目标：如"Q2重复错误率降低50%"
- ⚠️ 亚俊氏适用性：不推荐作为主框架，可作为补充

## 2. 错误分级与量化模型

### 2.1 三级错误分级（管理者原始需求）

| 等级 | 定义 | 示例 | 考核影响 |
|------|------|------|---------|
| 严重（Critical） | 造成重大经济损失或品牌损害，或涉及法规合规 | 库存决策导致数十万损失；产品认证缺失 | 直接影响年终奖（扣20-50%） |
| 次要（Major） | 造成可衡量的效率损失或客户体验下降 | 说明书单位错误；交期延误决策 | 影响绩效评分（扣10-20%） |
| 一般（Minor） | 造成轻微影响，可快速修复 | 文档格式不规范；内部流程小偏差 | 记录但不直接扣分（累计3次=1次次要） |

### 2.2 FMEA-RPN量化法（借鉴制造业）
每个错误评估三个维度（1-10分）：
- S（严重度）：错误造成的影响有多大
- O（发生频率）：这类错误多常发生
- D（可检测性）：现有流程能否提前发现（分数越高=越难发现）

RPN = S × O × D（范围1-1000）

| RPN范围 | 对应等级 | 行动 |
|---------|---------|------|
| 200+ | 严重 | 立即整改+考核扣分 |
| 80-199 | 次要 | 限期整改+记录 |
| <80 | 一般 | 记录+观察 |

### 2.3 重复错误加权
重复错误应加重处理：
- 首次发生：按原始等级计分
- 6个月内重复：等级自动上调一级（一般→次要，次要→严重）
- 12个月内第三次：无论原始等级，直接标记为严重

## 3. 多层责任分配与考核

### 3.1 责任矩阵与考核权重

```
错误发生
  ├── 执行层（工程师）
  │     └── 考核项：执行错误次数、文档质量分
  ├── 管理层（部门经理）
  │     └── 考核项：审核拦截率、下属重复错误率、团队错误趋势
  └── 决策层（副总）
        └── 考核项：分管领域错误总量、重大决策失误、人员管理有效性
```

### 3.2 层级考核指标设计

**执行层（工程师）**
| 指标 | 权重 | 计算方式 |
|------|------|---------|
| 个人错误次数（加权） | 50% | Σ(错误等级分值) |
| 重复错误率 | 30% | 重复错误数/总错误数 |
| 改进率 | 20% | (上期错误-本期错误)/上期错误 |

**管理层（部门经理）**
| 指标 | 权重 | 计算方式 |
|------|------|---------|
| 团队错误总量（加权） | 30% | 团队Σ(错误等级分值) |
| 审核拦截率 | 30% | 经理层拦截数/总错误数 |
| 下属重复错误率 | 20% | 下属重复错误/下属总错误 |
| 团队改进趋势 | 20% | 季度环比变化 |

**决策层（副总）**
| 指标 | 权重 | 计算方式 |
|------|------|---------|
| 分管领域重大错误数 | 25% | 严重级错误次数 |
| 个人决策失误 | 25% | 副总本人的决策错误 |
| 管理有效性 | 25% | 下属团队整体错误趋势 |
| 人员管理 | 25% | 是否及时处理不称职下属 |

## 4. 薪酬联动模型

### 4.1 扣分制（适合亚俊氏）

基础模型：年终奖 = 基准奖金 × 绩效系数

绩效系数计算：
```
绩效系数 = 1.0 - 错误扣分总和

其中：
  严重错误：每次扣 0.10-0.20
  次要错误：每次扣 0.03-0.05
  一般错误：每次扣 0.01
  重复错误：扣分翻倍

绩效系数下限 = 0.3（保底30%年终奖）
绩效系数上限 = 1.2（零错误+有突出贡献可超额）
```

### 4.2 示例计算

某副总年度错误记录：
- 严重错误1次（库存决策）：-0.15
- 次要错误2次（说明书问题）：-0.03 × 2 = -0.06
- 其中1次为重复错误：额外 -0.03
- 绩效系数 = 1.0 - 0.15 - 0.06 - 0.03 = 0.76

若基准年终奖20万：实发 20 × 0.76 = 15.2万

### 4.3 工资升降联动

| 年度绩效系数 | 薪酬影响 |
|-------------|---------|
| ≥1.0 | 有资格加薪5-10% |
| 0.8-0.99 | 薪酬不变 |
| 0.6-0.79 | 警告，连续两年则降薪5% |
| <0.6 | 降薪10%或调岗 |
| 连续3年<0.7 | 劝退/解聘评估 |

## 5. 集体决策记录与问责（借鉴党组织）

### 5.1 决策会议记录制度
管理者明确提到党委决策的优点：7人表态、记录存档、少数服从多数。

系统化实现：
| 要素 | 实现方式 |
|------|---------|
| 参会人员 | 系统记录所有参与者 |
| 议题与背景 | 结构化录入决策议题 |
| 每人表态 | 支持/反对/保留，附理由（文字或语音转文字） |
| 表决结果 | 自动统计多数意见 |
| 执行责任人 | 指定执行人和时间节点 |
| 执行追踪 | 定期检查执行偏差 |

### 5.2 执行纪律追踪
决策形成后，追踪执行偏差：
- 执行人是否按决策执行
- 是否有人公开表达与集体决策不一致的意见（"我当时就反对"）
- 执行结果与预期的偏差

## 6. 法律合规考量（中国劳动法视角）

### 6.1 关键合规点
| 风险 | 说明 | 对策 |
|------|------|------|
| 绩效考核合法性 | 《劳动合同法》要求考核标准事先告知 | 将考核制度写入员工手册，签字确认 |
| AI考核透明度 | 员工有权知道考核依据 | 每次扣分需有具体错误记录可查 |
| 降薪合法性 | 单方降薪需协商一致或有制度依据 | 考核制度需经民主程序（职工代表大会） |
| 解聘风险 | "不能胜任工作"需有培训/调岗记录 | 系统记录培训和改进机会 |
| 隐私保护 | 《个人信息保护法》约束 | 仅采集工作相关决策数据，不涉及个人隐私 |

### 6.2 合规实施建议
1. 考核制度经职工代表大会讨论通过
2. 每位员工签署知情同意书
3. 错误记录可申诉，设申诉流程
4. AI评估结果需人工复核确认
5. 保留完整的培训和改进记录

## 7. 推荐组合（适合亚俊氏）

| 方法论/模型 | 用途 | 优先级 |
|------------|------|--------|
| KPI + 决策质量指标 | 绩效考核主框架 | 必选 |
| 三级错误分级 | 错误严重度分类 | 必选 |
| 扣分制薪酬联动 | 奖金/工资挂钩 | 必选 |
| 层级考核指标体系 | 多层责任分配 | 必选 |
| 重复错误加权 | 重复犯错加重处罚 | 必选 |
| 决策会议记录制度 | 集体决策存档 | 推荐（第二阶段） |
| FMEA-RPN量化 | 精细化错误评分 | 推荐（成熟后引入） |
| BSC平衡计分卡 | 全面绩效框架 | 可选（长期升级） |
| 合规三件套 | 制度+知情+申诉 | 必选（法律底线） |
