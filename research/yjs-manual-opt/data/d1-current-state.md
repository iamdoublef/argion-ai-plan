## 三、发现的核心问题

### 问题1：一份文档服务多品牌，靠人工删减
文档末尾同时列出三个品牌的信息：
- WEVAC TECHNOLOGY CO., LIMITED（香港）
- Precision Appliance Technology, Inc.（美国Vesta）
- ADVANCED CUISINE TECHNOLOGY PTE. LTD.（新加坡）

注释写明"以下请按对应客户选择删减"——说明每次出说明书要人工处理，容易出错，也无法追溯。

### 问题2：关键字段未填写
保修卡中客服邮箱字段为占位符：`（请按需填写客服邮箱地址）`，说明文件在交付前还有手动填写步骤，存在遗漏风险。

### 问题3：安全警示体系不统一
- V23：WARNING + CAUTION 两级
- IMT050：WARNING + CAUTION + NOTICE 三级
同一家公司两个产品标准不一致，出海合规存在风险。

### 问题4：参数表是图片形式（部分）
食物保鲜期限对照表等数据内容以图片形式存在，多语言版本需要重新制作图片，维护成本高。

## 四、图片分类与处理建议

### 第一类：产品结构图 / 操作步骤图（主体）
- 特征：产品特定，与外观强绑定，专业线稿
- 现状：已提取至 `images_v23/`、`images_imt050/` 目录
- 建议：建图片库，按产品归档，多品牌共用（图中不含品牌信息）
- AI能做：❌ 无法生成准确技术线稿
- 来源：设计师 / CAD导出 / 工厂提供

### 第二类：警示图标 / 通用符号
- 特征：行业标准图标（WARNING三角、CE标志等）
- 建议：建图标库，一次建好永久复用
- AI能做：✅ 可以生成标准SVG图标

### 第三类：参数表 / 对照表
- 特征：本质是数据，以图片形式存在
- 建议：改为文字表格，由AI直接生成
- AI能做：✅ 完全可以生成和维护

## 五、AI可以做的 vs 不能做的

| 环节 | AI能做 | 备注 |
|------|--------|------|
| 内容生成（安全须知、操作步骤、故障排除） | ✅ | 需要产品规格作为输入 |
| 中→英翻译 | ✅ | 需配套术语表保证一致性 |
| 多品牌变体生成 | ✅ | 配置化后一键生成 |
| 参数表生成 | ✅ | 结构化数据输入即可 |
| 产品结构图 / 操作步骤图 | ❌ | 需设计师或CAD导出 |
| 图片上的多语言标注（SVG格式） | ✅ | 前提是图片改为SVG，文字层分离 |
