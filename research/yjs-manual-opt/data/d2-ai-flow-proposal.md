# d2-ai-flow-proposal — AI辅助说明书生产流程方案

日期：2026-02-25
状态：初稿，待大boss确认

---

## 目标流程

```
产品规格数据（结构化输入）
        ↓
   AI内容生成（文字层）
        ↓
  多品牌 / 多语言渲染
        ↓
  图片库调用（图片层）
        ↓
  Word / PDF 输出
```

---

## 三步落地路径

### 第一步：结构化产品信息（前提，1-2周）

做一个产品信息配置表（Excel或JSON），包含：
- 产品型号、功能列表、技术参数
- 品牌信息（名称、地址、网址、客服邮箱）—— 每个品牌一行
- 电压/频率规格（美标/欧标）
- 安全等级（适用标准：UL/CE/其他）

**效果**：变量从Word里剥离，改配置表而不改文档。

---

### 第二步：AI内容生成（核心）

输入产品配置 → Claude生成以下内容：
- 安全须知（统一三级体系：WARNING / CAUTION / NOTICE）
- 操作步骤（分功能模块）
- 故障排除
- 维护保养
- 参数表（文字表格，替代图片形式）
- 中→英翻译（配套术语表）
- 多品牌变体（自动替换品牌信息）

**预期效果**：初稿生成时间从2-3天 → 2-3小时。

---

### 第三步：图片库管理（图片层独立）

**命名规范**：
```
{产品型号}_{章节}_{序号}.png
示例：V23_structure_01.png
     V23_operation_seal_02.png
     IMT050_structure_01.png
```

**图片来源**（按优先级）：
1. 现有说明书提取（已完成：`images_v23/`、`images_imt050/`）
2. 设计师 / CAD导出（新产品）
3. 标准图标库（WARNING等通用符号）

**图文关联方式**：
文字模板中用占位符引用图片，排版时自动填入：
```
[图片: V23_structure_01]
说明：产品各部件名称标注图
```

**多语言图片处理**：
- 短期：图片内不含文字，标注放在图片下方的文字层
- 中期：改用SVG格式，文字层与图形层分离，支持多语言替换

## 难点与边界

| 难点 | 程度 | 解决方案 |
|------|------|---------|
| 产品技术线稿生成 | ❌ AI无法做 | 设计师/CAD，建库复用 |
| 图片内文字多语言 | ⚠️ 需改格式 | SVG分层（中期目标） |
| 合规审核（UL/CE） | ⚠️ 需人工确认 | AI生成初稿，专业人员审核 |
| 术语一致性 | ✅ 可管理 | 建术语表，AI翻译时引用 |
