# Swiss New Product Structure Draft Prompt

用途：
- 从简版 Word、产品图片和参数资料生成 Swiss 说明书的结构化草稿
- 输出只能是结构化 JSON 草稿或结构说明，不得输出最终成品文档

输入要求：
- 产品事实资料
- 简版源稿 Word/PDF 文本
- 图片清单与图片用途说明
- 适用品牌与区域
- 目标章节框架要求

硬规则：
- 以源稿事实为准，不得为了排版改写事实
- 章节结构必须符合 Swiss A5 共用母版的 block schema
- 程序性图片必须绑定到具体步骤或具体 block
- 遇到无法确定的内容，必须显式标记 `needs_review`
- 不得输出 HTML、CSS、富文本或最终 PDF 口吻

输出要求：
1. `chapter manifest` 草稿
2. 每章 `blocks` 草稿
3. 所有可翻译文本的 `text_id`
4. 图片绑定建议
5. 缺失信息清单
6. 需要人工确认的风险点

自检清单：
- 是否遗漏安全须知、操作步骤、保修信息等核心内容
- 是否有图片未绑定用途
- 是否把同一张程序性图片绑定到多个步骤
- 是否出现无法落到现有 block type 的内容
- 是否有需要新增模块而未标记的内容
