# 竞品分析方法论适配性评估

> 评估日期：2026-02-16
> 团队画像：2人AI团队，技术能力强（爬虫/API/数据分析/AI），无品牌运营/市场调研内部能力
> 数据获取方式：网络采集 + 购买第三方数据
> 评估目标：从20+方法论中筛选出"2人AI团队能落地执行"的组合

---

## 一、团队能力画像

### 能做的
- 网页爬虫（Playwright/Puppeteer）
- API对接（Amazon SP-API、第三方工具API）
- 数据清洗、分析、可视化
- AI/LLM驱动的文本分析（情感分析、主题提取、分类）
- 自动化流程搭建
- Dashboard开发

### 做不了的
- 英文用户访谈（无市场调研能力）
- 品牌运营执行（无品牌团队）
- 线下渠道调研（无渠道资源）
- 内部运营数据获取（非Argion员工，无ERP/CRM访问权）
- 大规模问卷调研（无受访者招募渠道）

### 可购买的
- Jungle Scout / Helium 10（$30-100/月）
- Keepa API（$15-20/月）
- SimilarWeb（$100+/月）
- 行业报告（$2K-5K/份）
- Amazon Brand Analytics（免费，但需品牌注册权限）

## 二、20+方法论逐一适配性评估

### 评估维度说明

| 维度 | 含义 |
|------|------|
| 可执行性 | 2人AI团队能不能独立完成？ |
| 数据可获取性 | 需要的数据能不能从外部获取？ |
| AI自动化程度 | 能不能用AI/自动化替代人工？ |
| 决策价值 | 产出对Argion发展策略有多大帮助？ |
| 综合适配度 | 综合评分（A/B/C/D/F） |

### 2.1 经典战略框架

#### Porter五力（轻量版）

| 维度 | 评估 |
|------|------|
| 可执行性 | ✅ 可以。基于公开数据（行业报告摘要、Amazon竞争格局、供应商信息）即可完成轻量版 |
| 数据可获取性 | 中。行业报告需购买完整版才有精确数据，但公开摘要+Amazon数据可支撑框架性分析 |
| AI自动化程度 | 中。数据采集可自动化，但五力判断需人工校验 |
| 决策价值 | 中。提供行业全局视角，避免只盯直接竞品。但对日常运营决策帮助有限 |
| 综合适配度 | B — 值得做轻量版，不值得深做 |
| 有效性判据 | 产出的行业结构判断能否被后续市场数据验证（如"替代品威胁高"→是否真的出现替代品） |

#### SWOT

| 维度 | 评估 |
|------|------|
| 可执行性 | ✅ 可以。但SWOT的价值取决于输入质量，外部团队做的SWOT容易流于表面 |
| 数据可获取性 | 低-中。S/W需要内部数据（成本结构、团队能力），外部团队只能基于公开信息推断 |
| AI自动化程度 | 高。AI可自动生成初稿，但质量存疑 |
| 决策价值 | 低。SWOT太泛，不直接指导行动 |
| 综合适配度 | D — 不推荐单独使用，可作为其他框架的辅助 |
| 有效性判据 | N/A（SWOT本身不产出可验证的结论） |

#### 价值链分析

| 维度 | 评估 |
|------|------|
| 可执行性 | ❌ 不可行。需要Argion内部运营数据（成本结构、生产流程、供应链细节） |
| 数据可获取性 | 极低。核心数据全在企业内部 |
| AI自动化程度 | 低 |
| 决策价值 | 高（如果能做的话） |
| 综合适配度 | F — 外部团队无法执行，需Argion内部配合 |
| 前置条件 | Argion提供成本结构、生产流程、供应链数据 |

#### 蓝海ERRC网格

| 维度 | 评估 |
|------|------|
| 可执行性 | ✅ 可以。基于已有的价值曲线分析直接推导 |
| 数据可获取性 | 高。已有数据足够 |
| AI自动化程度 | 高。AI可基于价值曲线自动生成ERRC建议 |
| 决策价值 | 高。直接产出"做什么/不做什么"的行动框架 |
| 综合适配度 | A — 零成本高价值，必做 |
| 有效性判据 | ERRC建议是否被产品团队采纳并执行；执行后市场表现是否改善 |

#### BCG矩阵 / GE-McKinsey

| 维度 | 评估 |
|------|------|
| 可执行性 | 部分可行。可基于Amazon数据做简化版（评价增速=市场增长率，BSR=相对份额） |
| 数据可获取性 | 中。精确市场份额数据缺失，只能用代理指标 |
| AI自动化程度 | 高 |
| 决策价值 | 中。对产品组合决策有参考价值 |

### 2.3 数据驱动方法

#### JTBD（Jobs-to-be-Done）

| 维度 | 评估 |
|------|------|
| 可执行性 | ⚠️ 有限。标准JTBD需要用户访谈（做不了），但可以用Amazon评价+Q&A做"数据驱动的JTBD近似" |
| 数据可获取性 | 中。Amazon评价文本可采集，但深度不如访谈 |
| AI自动化程度 | 高。LLM可从评价文本中提取用户任务和痛点 |
| 决策价值 | 极高。理解"客户为什么买"是最关键的缺失环节 |
| 综合适配度 | B+ — 不能做标准版，但AI驱动的近似版非常值得做 |
| 有效性判据 | 提取的"客户任务"是否与实际购买行为一致（可通过A/B测试Listing文案验证） |
| 具体做法 | 爬取6品牌Top 500评价+全部Q&A → LLM提取"用户想完成的任务"和"选择/放弃的原因" → 聚类为JTBD地图 |

#### Kano模型

| 维度 | 评估 |
|------|------|
| 可执行性 | ✅ 可以。标准Kano需问卷，但可用AI从评价数据中推断功能分类 |
| 数据可获取性 | 高。已有评价关键词数据+功能特性矩阵 |
| AI自动化程度 | 高。LLM可基于评价情感判断功能属于必备/期望/兴奋 |
| 决策价值 | 高。直接指导产品功能优先级 |
| 综合适配度 | A — 基于已有数据即可执行，高价值 |
| 有效性判据 | Kano分类结果是否与用户实际购买决策权重一致 |
| 具体做法 | 功能有→好评多=必备；功能有→好评少=无差异；功能有→惊喜评价=兴奋；功能无→差评多=必备缺失 |

## 四、推荐执行方案：2人AI团队最优组合

### 4.1 修正后的框架（从五层精简为四层）

原推荐的五层框架中，Layer 0（Porter五力）对日常决策帮助有限，降为可选项。修正为：

```
Layer 1: 产品竞争对标（已完成80%）
  特性矩阵 + Kano分类(AI版) + 价值曲线 + ERRC网格
  → 产出：功能优先级矩阵、ERRC行动框架

Layer 2: 客户需求洞察（当前最大缺口）
  VoC深度分析 + JTBD近似(AI版)
  → 产出：用户痛点排名、客户任务地图、品牌切换原因

Layer 3: 电商竞争情报（需要工具投入）
  Jungle Scout + Keepa + ABA(如有权限)
  → 产出：销量趋势、搜索份额、定价策略、关键词竞争地图

Layer 4: 行动转化
  Battlecard + 90天路线图
  → 产出：每个竞品一页作战卡、优先级行动清单
```

### 4.2 执行计划

**Phase 1: 零成本补强（1-2周）**

| 任务 | 输入 | 输出 | 工具 | 工时 |
|------|------|------|------|------|
| VoC深度分析 | 6品牌各Top 300评价文本 | 情感分析报告+痛点排名+跨品牌对比 | Playwright爬取 + LLM分析 | 3天 |
| Kano功能分类 | 已有特性矩阵+评价数据 | 功能分类表（必备/期望/兴奋/无差异） | LLM分类 | 1天 |
| JTBD近似分析 | 评价文本+Q&A | 客户任务地图+品牌切换原因 | LLM提取+聚类 | 2天 |
| ERRC网格 | 价值曲线数据 | 消除/减少/增加/创造四象限 | LLM生成+人工校验 | 0.5天 |
| Battlecard | 全部已有数据 | 6张竞品作战卡 | LLM模板化生成 | 1天 |

### 4.3 有效性验证机制

每个方法论的产出都需要验证，不能"做了就算有效"：

| 方法论 | 验证方式 | 验证时间点 |
|--------|---------|-----------|
| VoC分析 | 痛点排名是否与产品改进后的评分变化一致 | 产品改进后3个月 |
| Kano分类 | 新增"兴奋功能"是否带来评分/转化率提升 | 功能上线后1个月 |
| JTBD | 基于JTBD优化的Listing文案是否提升转化率 | A/B测试2周 |
| ERRC | 执行ERRC建议后市场表现是否改善 | 执行后3-6个月 |
| Battlecard | 销售团队是否实际使用 | 发布后1个月 |
| Jungle Scout | 销量预估与实际的偏差率 | 持续对比 |
| Keepa | 定价策略调整后销量是否符合预期 | 调价后2周 |

## 五、方法论有效性的根本判据

方法论是否有效，最终只有一个标准：**产出的结论是否导致了更好的商业决策。**

具体来说：
1. 分析结论是否被决策者采纳？（如果没人看，再好的分析也无效）
2. 基于分析做出的决策，结果是否优于不做分析时的决策？（需要对照组或历史对比）
3. 分析的投入产出比是否合理？（花2周分析省了2个月的试错，就是有效的）

对于2人AI团队，最务实的有效性判据是：
- Phase 1产出物（VoC/Kano/JTBD/ERRC/Battlecard）是否被Argion团队认为"有用"
- Phase 2数据（Jungle Scout/Keepa）是否改变了某个具体决策
- 6个月后回看，哪些分析结论被市场验证了，哪些被证伪了

## 六、与methodology-benchmark.md的关系

`methodology-benchmark.md`（656行）是方法论的"百科全书"——20+方法论的理论介绍和通用评估。

本文件是"落地手册"——基于2人AI团队的具体约束，筛选出能执行的方法论组合，给出具体执行计划和验证机制。

两者的关系：benchmark是"菜单"，本文件是"点单"。
