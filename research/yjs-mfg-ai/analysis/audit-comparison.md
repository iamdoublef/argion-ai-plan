## 3. P0/P1/P2 分组逻辑检查

### P0（立即启动）：A5 + B5 — PASS

- 两者均为纯SaaS工具，无硬件改造，投入均<12万元
- 实施周期以周计，风险极低，沉没成本可忽略
- A5 ROI排名#1（~48x），B5排名#4（~2.4x），均为高ROI场景

### P1（6个月内启动）：B2路径A + B1国产 + B4方案A — PASS

- 三者技术成熟度均达4/5或核心模块4/5
- 投入30-300万元区间，回收期均在2年以内
- B2路径A ROI最高(~2.8x)，B4方案A投入最低(30-50万)，B1效果确定性最高

### P2（12个月后评估）：A1 + A2 + A3 + A4 + B3 + B2路径B/C — PASS

- A1/A2：全面部署投入大（500万+），需先POC验证
- A3：数据不足无法计算ROI，需先完成数据准备
- A4：数据瓶颈（需50-200+条配方数据），ROI虽高(~2.6x)但可行性受限
- B3：年直接收益偏低（29-90万），核心价值在间接影响（难量化）
- B2路径B/C：投入大/周期长，建议路径A验证后再升级

分组逻辑自洽：P2中A4方案B的ROI(~2.6x)高于多个P1场景，但因数据瓶颈放入P2，基于可行性而非纯ROI排序，决策合理。

## 4. WARNING清单

### W-01：A5投入人民币折算低端偏高

- 位置：总览对比表 A5行 + P0表
- 问题：源文件标注$3,000-$6,000/年。按汇率7.2折算应为2.16-4.32万元，报告写"3-4.3万元"，低端从2.16万上调至3万（偏差+39%）
- 影响：低估A5成本优势，但A5 ROI极高（~48x），对排名和决策无实质影响
- 建议：将"3-4.3万"修正为"2.2-4.3万"

### W-02：B3技术成熟度拆分表述与源文件不完全对应

- 位置：总览对比表 B3行
- 问题：源文件技术成熟度评分表给出综合"4"分（技术成熟度维度），未单独给出"传感器+规则"和"AI模型"的分拆评分。报告的"4/5（传感器+规则）；3/5（AI模型）"是合理推断但非源文件原文
- 影响：表述合理，反映了两种方案的实际成熟度差异
- 建议：可保留，建议加注"综合源文件多维度评分推断"

### W-03：ROI排名#6和#7顺序与精确计算略有偏差

- 位置：ROI排名表 第6-7行
- 问题：精确计算B2-C(1.48x) > B3-A(1.44x)，但报告中B3-A(#6)排在B2-C(#7)前面
- 影响：差异仅0.04x，三者均在~1.5x水平，四舍五入后无法区分，不影响分组决策
- 建议：可保留，或加注"#5-#7 ROI接近，排名可互换"

### W-04：回收期中位数计算方法不一致

- 位置：ROI排名表 "回收期中位数" 列
- 问题：部分场景使用源文件声明的回收期区间中位数（如B5取6-12个月中位数9个月），部分场景使用投入/收益直接计算（如A4方案B：87.5/230x12=4.6个月，报告写5个月，而源文件声明6-18个月）
- 影响：A4方案B的5个月 vs 源文件6-18个月存在差异，但源文件的6-18个月是方案A+B合并区间，单独计算方案B有其合理性
- 建议：统一计算方法，或在表头注明计算口径

### W-05：A5投入中位数与总览表低端不一致

- 位置：ROI排名表 A5行 vs 总览对比表 A5行
- 问题：排名表写"3.2（$4.5Kx7.2/10）"，中位数3.24万计算正确。但总览表低端写"3万"（应为2.16万），两处口径不一致
- 影响：与W-01同源，对决策无实质影响
- 建议：与W-01一并修正
